# HU001 — Necesito generar usuarios, roles y accesos

**Identificador:** HU001

## Historia de Usuario

> **Como** Administrador,  
> **quiero** generar usuarios, roles y accesos,  
> **para** controlar quien puede entrar al sistema y que puede hacer.

## Criterios de Aceptación

| # | Criterio | Contexto | Evento | Resultado Esperado |
|---|----------|----------|--------|--------------------|
| 1.0 | Crear usuarios | Se necesita registrar un nuevo usuario con un rol específico | El administrador ingresa al módulo de usuarios y selecciona “Crear usuario” | El sistema permite ingresar los datos y genera correctamente el nuevo usuario |
| 2.0 | Asignar, modificar o retirar roles | Un usuario ya existe y requiere ajustes de permisos | El administrador selecciona un usuario y edita sus roles | El sistema actualiza los roles asignados y los cambios se reflejan de inmediato |
| 3.0 | Recuperación de contraseña | El usuario no puede acceder por olvido de credenciales | El usuario solicita recuperar contraseña | El sistema envía instrucciones seguras para restablecerla |
| 4.0 | Solicitudes de reactivación | Un usuario previamente inactivo requiere nuevamente acceso | El administrador revisa y aprueba la solicitud | El sistema reactiva la cuenta y notifica al usuario |
| 5.0 | Validar inicio de sesión | El usuario intenta ingresar con sus credenciales | El usuario envía correo y contraseña | El sistema valida la información y permite o rechaza el acceso |
| 6.0 | Registrar actividad de accesos | Usuarios acceden o cierran sesión | Se genera un evento de acceso | El sistema registra la actividad para fines de auditoría |
