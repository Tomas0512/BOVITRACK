# RF030 — Auditoria de Actividades

| Campo | Valor |
|-------|-------|
| **Código** | RF030 |
| **Nombre** | Auditoria de Actividades |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe llevar un registro automático de todas las acciones realizadas por los usuarios. Esto incluye, entre otras: inicios de sesión, creación y edición de animales, movimientos de inventario, registro de vacunas, exportación de reportes y cambios de permisos. Cada acción debe quedar registrada de manera automática, sin intervención del usuario.

## Controles y Restricciones

- Control de obligatoriedad: Cada registro de auditoría debe contener obligatoriamente usuario, fecha, hora y acción realizada.
- Control de integridad: Los registros de auditoría no pueden ser alterados después de guardarse.
- Control de acceso: El sistema debe permitir filtrar el registro de auditoría por usuario, módulo y rango de fechas.
- Restricción de permisos: Solo el administrador puede consultar el registro completo de auditoría.
- Restricción de acceso: Los empleados solo pueden ver su propio historial de actividades, no el de otros usuarios.
- Restricción de integridad: Los registros de auditoría no se pueden eliminar ni modificar.
- Restricción funcional: El sistema debe conservar los registros por un tiempo mínimo definido por la finca.

## Criterios de Aceptación

- Cada acción queda registrada automáticamente con trazabilidad completa.
- El administrador puede consultar la auditoría aplicando filtros por usuario, fecha o tipo de acción.
- El sistema debe permitir generar reportes de auditoría en PDF o Excel, con encabezado de finca, fecha de generación y usuario solicitante.
- Si un empleado consulta la auditoría, únicamente puede ver su propio historial de actividades.
- Los registros de auditoría deben presentarse en un formato claro y entendible para el usuario.
- Ningún usuario puede modificar ni eliminar los registros de auditoría.
