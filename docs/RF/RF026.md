# RF026 — Control de Inventario

| Campo | Valor |
|-------|-------|
| **Código** | RF026 |
| **Nombre** | Control de Inventario |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe administrar el inventario de insumos utilizados en la finca, incluyendo medicamentos, vacunas, suplementos alimenticios, concentrados y equipos. Cada insumo debe registrarse con: nombre, categoría, unidad de medida (kg, litros, dosis, bultos), fecha de vencimiento (si aplica), cantidad en stock, costo unitario y proveedor. Los movimientos de inventario pueden ser: Ingreso (compra o donación). Consumo (uso en animales, lotes o potreros). Ajuste (pérdidas, mermas o correcciones).

## Controles y Restricciones

- Control de stock mínimo: El sistema debe generar alertas cuando el inventario de un insumo llegue al nivel mínimo configurado.
- Control de fechas: Los insumos con fecha de vencimiento no pueden ser aplicados después de expirar.
- Control de consumo: No se puede registrar consumo de insumos sin stock disponible suficiente.
- Control de obligatoriedad: Todos los campos críticos (nombre, categoría, cantidad, unidad de medida) deben estar diligenciados antes de guardar.
- Restricción de permisos: Solo administradores y empleados autorizados podrán registrar ingresos, consumos y ajustes de inventario.
- Restricción de coherencia: No se pueden registrar cantidades negativas en stock.
- Restricción temporal: No se permite registrar consumos con fecha futura.

## Criterios de Aceptación

- El sistema impide registrar un consumo si la cantidad solicitada supera el stock disponible.
- Se generan reportes de kardex (entradas, salidas y existencias) con fechas y usuarios responsables.
- Cuando un insumo llega a su stock mínimo, el sistema muestra una alerta visible en el panel.
- El historial de cada insumo debe reflejar todos sus movimientos (ingresos, consumos, ajustes), con cantidades, costos y usuarios.
- El sistema bloquea la aplicación de vacunas o medicamentos vencidos, notificando al usuario el error.
