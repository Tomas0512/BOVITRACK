# RF023 — Registro Productivo y Economico

| Campo | Valor |
|-------|-------|
| **Código** | RF023 |
| **Nombre** | Registro Productivo y Economico |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe registrar todos los movimientos productivos y económicos de la finca, incluyendo: Gastos: alimentación, medicamentos, suplementos, servicios, mano de obra, energía, mantenimiento. Ingresos: venta de leche, venta de animales, venta de subproductos. Asignación de costos: cada gasto o ingreso debe poder asignarse a nivel de finca, lote o animal específico cuando aplique. El sistema debe calcular automáticamente indicadores como: costo por litro de leche, costo por kilo ganado de engorde y margen de rentabilidad por lote.

## Controles y Restricciones

- Control de obligatoriedad: Cada movimiento debe registrar fecha, tipo (ingreso o gasto), monto, centro de costo y comprobante.
- Control de valores: El monto debe ser un valor positivo y con moneda definida (COP).
- Control de consistencia: No se pueden asignar gastos a animales o lotes inactivos.
- Restricción de permisos: Solo el administrador o empleados autorizados pueden registrar ingresos y gastos.
- Restricción de integridad: Los registros no se pueden borrar, solo anular con una justificación.
- Restricción funcional: Cada gasto debe pertenecer a una categoría definida (alimentación, sanidad, etc.).

## Criterios de Aceptación

- Al registrar un gasto, el sistema debe reflejarlo inmediatamente en los reportes de costos de la finca/lote correspondiente.
- El sistema debe impedir guardar un movimiento sin categoría, fecha o monto definido.
- Los reportes económicos deben mostrar de manera clara: total de ingresos, total de gastos y rentabilidad por período.
- Al consultar un animal o lote, el sistema debe mostrar su costo acumulado y la participación en la rentabilidad general.
- Cada registro económico debe quedar en el historial con usuario, fecha, monto, categoría y documento soporte.
- Si se intenta asignar un gasto a un animal inactivo, el sistema debe bloquear la acción y notificar el error.
