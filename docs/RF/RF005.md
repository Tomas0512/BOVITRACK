# RF005 — Asignación de Roles a Usuarios

| Campo | Valor |
|-------|-------|
| **Código** | RF005 |
| **Nombre** | Asignación de Roles a Usuarios |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir que el administrador asigne, modifique o revoque roles a los usuarios registrados. La asignación de roles debe ser flexible, permitiendo actualizar los permisos de un usuario en cualquier momento según sus funciones en la finca.

## Controles y Restricciones

- Control de obligatoriedad: No se puede guardar un usuario sin un rol asignado.
- Control de cambios: Cada cambio de rol debe registrarse con fecha, hora, usuario responsable y motivo.
- Restricción de permisos: Solo el administrador puede asignar o cambiar roles.
- Restricción de consistencia: No se pueden asignar roles inexistentes o eliminados.
- Restricción de seguridad: El administrador no puede retirarse a sí mismo el rol de administrador sin asignar otro previamente.
.

## Criterios de Aceptación

- Al crear un usuario, el sistema obliga a seleccionar un rol válido.
- Cada vez que un rol de usuario se modifique, se debe generar automáticamente un registro en el historial con trazabilidad completa.
- Si se intenta asignar un rol inexistente, el sistema bloquea la acción y muestra un mensaje de error.
- El sistema permite consultar en todo momento qué rol tiene asignado cada usuario y el historial de cambios de roles.
- Los cambios en roles se reflejan inmediatamente en el acceso y visibilidad de módulos del usuario.
