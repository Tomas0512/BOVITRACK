# RF028 — Gestión de Usuarios y Roles

| Campo | Valor |
|-------|-------|
| **Código** | RF028 |
| **Nombre** | Gestión de Usuarios y Roles |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir que el administrador gestione usuarios y defina los roles disponibles en la finca. Para cada usuario se deben registrar datos básicos (nombre, correo electrónico, finca asociada, estado de la cuenta) y asignarle un rol predeterminado (administrador, capataz, empleado u otros definidos por el administrador). Cada rol debe contar con permisos diferenciados y configurables según los módulos del sistema.

## Controles y Restricciones

- Control de rol: El sistema valida que cada usuario tenga siempre un rol asignado.
- Control de unicidad: No se permiten correos electrónicos duplicados entre usuarios.
- Control de asociación: Cada usuario debe estar vinculado a una finca obligatoriamente.
- Control de permisos: El administrador define qué módulos están habilitados para cada rol.
- Restricción de permisos: Solo el administrador puede crear, modificar o eliminar usuarios y roles.
- Restricción de acceso: Un usuario no puede acceder a fincas a las que no esté asignado.
- Restricción funcional: No se permite iniciar sesión si la cuenta está inactiva o suspendida.
- Restricción de seguridad: Un rol no puede quedar sin permisos mínimos (ejemplo: inicio de sesión).

## Criterios de Aceptación

- El sistema bloquea la creación de usuarios con correos duplicados y muestra un mensaje claro de error.
- El administrador puede crear nuevos roles o modificar permisos de los existentes, y los cambios se reflejan inmediatamente en el sistema.
- Al iniciar sesión, el usuario solo ve las funciones que le corresponden según su rol.
- El historial registra todas las creaciones, modificaciones y eliminaciones de usuarios y roles, indicando quién realizó la acción y cuándo.
- Si un usuario es desactivado, ya no puede iniciar sesión, pero su historial de actividades se mantiene intacto.
