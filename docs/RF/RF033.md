# RF033 — Gestión de Proovedores y Clientes

| Campo | Valor |
|-------|-------|
| **Código** | RF033 |
| **Nombre** | Gestión de Proovedores y Clientes |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir registrar y gestionar proveedores (alimentación, medicamentos, insumos) y clientes (compradores de leche, carne o animales). Para cada proveedor o cliente se deben registrar datos como: nombre o razón social, tipo (proveedor/cliente), identificación tributaria, dirección, teléfono, correo electrónico y observaciones. Los proveedores podrán vincularse a compras de insumos y servicios, y los clientes a ventas de productos o animales.

## Controles y Restricciones

- Control de obligatoriedad: No se podrán guardar registros sin nombre, tipo y número de identificación tributaria(NIT).
- Control de unicidad: No se permitirá registrar dos proveedores o clientes con la misma identificación tributaria.
- Restricción de permisos: Solo el administrador o empleados autorizados podrán crear, modificar o eliminar proveedores/clientes.
- Restricción de consistencia: Un proveedor solo puede vincularse a compras y un cliente solo a ventas.

## Criterios de Aceptación

- El sistema impide registrar un proveedor/cliente duplicado y muestra un mensaje de error claro.
- Cada proveedor/cliente queda vinculado a las transacciones económicas correspondientes.
- Los reportes muestran compras y ventas clasificadas por proveedor o cliente.
- Se puede consultar el historial de compras y ventas relacionadas a cada proveedor/cliente.
