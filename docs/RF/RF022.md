# RF022 — Manejo de Potreros/Pasturas

| Campo | Valor |
|-------|-------|
| **Código** | RF022 |
| **Nombre** | Manejo de Potreros/Pasturas |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir crear y gestionar potreros o pasturas dentro de la finca. Para cada potrero se deben registrar datos como: nombre o código, área total en hectáreas, capacidad de carga máxima (número de animales que puede soportar), estado de la cobertura (bueno, regular, sobrepastoreado), y fechas de ocupación y descanso. Además, debe ser posible asignar lotes de animales a potreros, registrar las fechas de ingreso/salida y llevar control del tiempo de descanso entre ocupaciones. El sistema debe generar alertas cuando se detecte sobrepastoreo o descanso insuficiente según parámetros configurados.

## Controles y Restricciones

- Control de capacidad: El sistema valida que la cantidad de animales asignados no supere la capacidad del potrero y que esta capacidad no sea cero o negativa..
- Control de fechas: Las fechas de ocupación y descanso no pueden solaparse con registros anteriores.
- Control de obligatoriedad: No se podrá guardar un potrero sin nombre, área o capacidad de carga definida.
- Restriccion de obligatoriead: Cada potrero debe estar obligatoriamente asociado a una finca registrada.
- Restriccion de Permisos:Solo el administrador o empleados autorizados pueden crear o modificar potreros.
- Restriccion de no duplicidad:Un lote no puede estar asignado simultáneamente a dos potreros en la misma fecha.

## Criterios de Aceptación

- El sistema debe impedir asignar un lote a un potrero cuya capacidad ya está superada.
- Los reportes de rotación deben mostrar con claridad cuántos días estuvo ocupado y descansando cada potrero.
- Al intentar guardar un potrero con datos incompletos, el sistema debe informar específicamente qué falta.
- El historial del potrero debe registrar todos los eventos de ocupación, descanso y asignación de lotes, con fecha, hora y usuario responsable.
- Cuando un potrero alcanza el límite de carga, el sistema debe mostrar una alerta inmediata al usuario.
- La interfaz debe permitir consultar el estado actual de cada potrero (libre, ocupado, en descanso) en tiempo real.
