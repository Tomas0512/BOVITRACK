# RF007 — Solicitud de Reactivacion de Cuentas de Usuarios

| Campo | Valor |
|-------|-------|
| **Código** | RF007 |
| **Nombre** | Solicitud de Reactivacion de Cuentas de Usuarios |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir que un usuario cuya cuenta esté inactiva o suspendida solicite su reactivación. La solicitud debe enviarse al administrador, quien tendrá la facultad de aprobar o rechazar la petición.

## Controles y Restricciones

- Control de estado: Solo usuarios inactivos pueden solicitar la reactivación.
- Control de obligatoriedad: La solicitud debe incluir motivo y justificación.
- Restricción de permisos: Solo el administrador puede aprobar o rechazar solicitudes.
- Restricción de integridad: Una cuenta no puede reactivarse automáticamente sin aprobación.
- Restricción de historial: Cada solicitud debe quedar registrada con fecha, hora, usuario solicitante y decisión del administrador.

## Criterios de Aceptación

- El sistema permite al usuario enviar una solicitud de reactivación desde la pantalla de inicio de sesión o portal de soporte.
- El administrador recibe una notificación en el sistema sobre la solicitud.
- El administrador puede aprobar o rechazar la solicitud, registrando observaciones.
- El usuario recibe una notificación con la decisión tomada.
- El historial conserva el registro de todas las solicitudes y decisiones relacionadas.
