# RF010 — Registro de Alimentación y Suplementación

| Campo | Valor |
|-------|-------|
| **Código** | RF010 |
| **Nombre** | Registro de Alimentación y Suplementación |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir que el capataz o empleado registre la alimentación de cada lote de animales en la finca, utilizando un control de stock por bulto o saco. La información a registrar incluye: Tipo de alimento suministrado (pasto, concentrado, silo, sal mineralizada, suplementos).  Cantidad utilizada por lote, indicando cuántos kilos se extrajeron de cada bulto o saco. Origen del alimento: identificación del bulto o saco de stock de donde se extrajo el alimento. Fecha y hora de suministro al lote.  Observaciones generales sobre consumo, apetito o comportamiento del lote. Este registro permite monitorear la alimentación efectiva, controlar el stock real de cada bulto, calcular raciones promedio por lote y generar reportes precisos de consumo de alimento.

## Controles y Restricciones

- Control de registro por lote: Cada registro debe estar vinculado a un lote previamente registrado en el sistema.Control de cantidades: La cantidad extraída de cada bulto debe ser positiva y coherente con el peso disponible en el bulto.Control de consistencia: El sistema puede comparar la cantidad total extraída con la cantidad disponible en stock para prevenir errores o inconsistencias.
- Control de fecha/hora: Cada registro debe incluir la fecha y hora de suministro para llevar trazabilidad.
- Control de stock de bultos: No se puede extraer más alimento de un bulto del que queda disponible en inventario.Restricción de permisos: Solo empleados autorizados o el administrador pueden registrar la alimentación; usuarios sin permisos no pueden acceder.
- Restricción de datos obligatorios: No se permite guardar registros incompletos, como tipo de alimento, cantidad extraída o identificación del bulto.

## Criterios de Aceptación

- El sistema permite registrar alimentación únicamente para lotes válidos y con bultos disponibles.
- Se notifican errores si faltan campos obligatorios o los valores ingresados no son válidos.
- Cada registro incluye fecha, hora, usuario que realizó el registro, cantidad extraída y bulto de origen.
- Los reportes muestran correctamente el consumo de cada lote y la reducción de stock de cada bulto.
- El historial de cada lote registra automáticamente todos los suministros de alimento para trazabilidad.
- No se permite extraer más kilos de un bulto que los disponibles en inventario.
