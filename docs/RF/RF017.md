# RF017 — Calendario de Eventos

| Campo | Valor |

| **Nombre** | Calendario de Eventos |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

Control de obligatoriedad: Cada evento debe registrar fecha, hora, tipo y responsable.    Control de fechas: No se permite registrar eventos con fecha pasada (salvo como histórico) ni fechas incoherentes.    Restricción de permisos: Solo administradores y capataces pueden crear, editar o eliminar eventos; los empleados solo pueden consultarlos.    Restricción de duplicidad: No se deben registrar dos eventos idénticos en la misma fecha y hora para el mismo lote o animal.

## Controles y Restricciones

- El sistema muestra en el calendario todos los eventos registrados por finca y permite filtrarlos por lote, animal o tipo de evento.
- Los usuarios reciben alertas automáticas por correo o WhatsApp de los eventos programados con la anticipación configurada.
- Si faltan datos obligatorios (ejemplo: fecha u hora), el sistema impide guardar y muestra un mensaje específico.
- Los reportes del calendario deben poder exportarse en PDF o Excel con la lista de eventos futuros y pasados.

## Criterios de Aceptación

- El sistema permite registrar eventos sanitarios únicamente para animales o lotes válidos.
- Se notifican errores si faltan campos obligatorios o los valores ingresados no son válidos.
- Cada registro incluye fecha, hora, usuario que realizó el registro, tipo de medicamento y dosis administrada.
- Los reportes muestran correctamente el historial sanitario de cada animal o lote, incluyendo vacunaciones, desparasitaciones y observaciones.
