# RF027 — Gestión de Documentos y Archivos Adjuntos

| Campo | Valor |
|-------|-------|
| **Código** | RF027 |
| **Nombre** | Gestión de Documentos y Archivos Adjuntos |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir cargar, almacenar y asociar documentos o archivos digitales relevantes para la gestión de la finca, como facturas, guías de movilización, certificados sanitarios, contratos, fotos de animales y soportes de compras o ventas. Cada archivo debe estar vinculado a un módulo (animal, lote, finca, transacción) y contar con metadatos básicos: tipo de documento, fecha, usuario que lo cargó y descripción opcional.

## Controles y Restricciones

- Control de obligatoriedad: Todo documento debe tener tipo, fecha y módulo asociado antes de guardarse.
- Control de formato: Solo se permiten formatos estándar como PDF, JPG, PNG, XLSX y DOCX.
- Control de tamaño: Los archivos no deben superar el límite definido por la finca (ejemplo: 10 MB por archivo).
- Restricción de permisos: Solo administradores y capataces pueden eliminar documentos; los empleados solo pueden visualizarlos.
- Restricción de duplicidad: No se permiten documentos con el mismo nombre y fecha en un mismo módulo.

## Criterios de Aceptación

- Los usuarios pueden consultar y descargar fácilmente los documentos vinculados a animales, lotes o fincas.
- Si un documento excede el tamaño permitido, el sistema muestra un mensaje de error y no permite la carga.
- Cada documento queda registrado en el historial con usuario, fecha y descripción.
- Los reportes incluyen un listado de documentos por tipo, fecha y módulo asociado.
