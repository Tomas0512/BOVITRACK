# RF031 — Modo Offline

| Campo | Valor |
|-------|-------|
| **Código** | RF031 |
| **Nombre** | Modo Offline |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir registrar datos sin conexión a internet y sincronizarlos automáticamente cuando la conexión se recupere. En caso de conflicto entre registros, el sistema debe advertirlo y permitir que el administrador decida qué dato conservar.

## Controles y Restricciones

- Control de integridad: Los registros hechos sin conexión deben guardarse con una marca de “pendiente de sincronización”.
- Control de sincronización: Al recuperar la conexión, los datos pendientes deben subirse automáticamente al sistema.
- Control de consistencia: El sistema debe informar al usuario si hubo errores en la sincronización.
- Restricción de permisos: Solo usuarios con cuenta activa pueden registrar datos en modo offline.
- Restricción de seguridad: Los datos almacenados localmente deben estar protegidos.
- Restricción funcional: En modo offline solo se permite registrar información; no se pueden generar reportes generales.
- Restricción temporal: Si un registro permanece sin sincronizar durante demasiado tiempo (ejemplo: 30 días), el sistema debe advertir al administrador.

## Criterios de Aceptación

- Los usuarios pueden registrar información sin internet y verla como “pendiente de sincronización”.
- Cuando vuelve la conexión, los datos pendientes se cargan automáticamente al sistema central.
- El sistema informa al usuario si algún dato no se sincronizó y permite reintentar.
- Cada sincronización queda registrada con fecha, hora, usuario y cantidad de datos procesados.
- Si hay conflictos entre datos, el sistema los marca claramente para que el administrador decida.
