# RF024 — Generacion de Reportes Productivos

| Campo | Valor |
|-------|-------|
| **Código** | RF024 |
| **Nombre** | Generacion de Reportes Productivos |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe generar reportes consolidados sobre la gestión productiva, económica y sanitaria de la finca. Estos reportes deben integrarse automáticamente con la información registrada en los módulos correspondientes (inventario de animales, reproducción, sanidad, ordeño, alimentación, pesajes, ingresos y gastos), sin necesidad de volver a capturar datos.  Los reportes deben estar disponibles en tres modalidades:
PDF: documentos listos para imprimir o compartir. Excel: archivos exportables para análisis detallado. Paneles interactivos en pantalla: con tablas dinámicas, gráficos y filtros configurables.

## Controles y Restricciones

- Control de filtros: El sistema debe requerir al menos finca y rango de fechas antes de generar un reporte.
- Control de integridad: Los datos de los reportes deben coincidir exactamente con los registros base de cada módulo.
- Control de exportación: Cada archivo generado debe contener fecha/hora de generación, finca y usuario solicitante.
- Control de seguridad: El contenido visible depende del rol del usuario (ejemplo: un empleado no ve reportes contables).
- Control de formato: El PDF debe mantener un diseño estándar con encabezados y pies de página; el Excel debe ser ordenable y filtrable.
- Restricción de permisos: Solo administradores y empleados autorizados podrán exportar reportes.
- Restricción de integridad: No se deben generar reportes vacíos; si no hay datos, el sistema debe mostrar un mensaje: “No se encontraron registros para el período seleccionado”.
- Restricción de coherencia: Los reportes solo deben reflejar información de las fincas a las que el usuario está asociado.
- Restricción funcional: No se pueden modificar los reportes generados desde el sistema; si hay errores, deben corregirse en los datos base y regenerar el reporte.

## Criterios de Aceptación

- Si el usuario intenta exportar sin definir rango de fechas, el sistema muestra un mensaje de error específico.
- Los reportes en PDF deben abrirse correctamente en cualquier lector estándar, incluyendo tablas y gráficos con el mismo formato de pantalla.
- Los reportes en Excel deben incluir filtros automáticos y permitir la manipulación de datos sin pérdida de estructura.
- Los paneles interactivos deben permitir aplicar filtros (finca, lote, período, categoría) sin necesidad de recargar toda la página.
- Cada exportación debe registrarse en el historial con fecha, hora, usuario, tipo de archivo y filtros aplicados.
- Los resultados de reportes (ejemplo: litros de leche por mes, costos por lote, indicadores sanitarios) deben coincidir al 100% con los datos provenientes de los módulos de origen
