# RF025 — Alertas Automáticas

| Campo | Valor |
|-------|-------|
| **Código** | RF025 |
| **Nombre** | Alertas Automáticas |
| **Versión** | 3.0 |
| **Prioridad** | Media |

## Descripción

El sistema debe generar alertas automáticas relacionadas con la gestión del hato, basadas en las fechas registradas y protocolos sanitarios o reproductivos. Entre las principales alertas se incluyen: Vacunas y refuerzos pendientes. Próximos partos estimados. Fechas de destete. Tratamientos médicos o vitamínicos programados. Rotación de potreros próxima. Las alertas deben mostrarse en el panel principal del sistema y, de manera opcional, enviarse por correo o WhatsApp. Cada alerta debe poder marcarse como “atendida” o “pospuesta”.

## Controles y Restricciones

- Control de programación: Cada evento debe generar una única alerta para la fecha definida, evitando duplicados.
- Control de estado: Una alerta no desaparece hasta ser marcada como atendida o cumplida.
- Control de configuración: El administrador podrá definir con cuántos días de anticipación se generan las alertas.
- Control de integridad: Las alertas deben estar siempre asociadas a un animal, lote o potrero existente.
- Restricción de permisos: Solo usuarios con acceso autorizado (administrador, capataz, empleados asignados) podrán visualizar y marcar alertas.
- Restricción temporal: No se deben generar alertas con fechas pasadas .
- Restricción de asociación: No se generan alertas para animales dados de baja del inventario.
- Restricción funcional: No se permite eliminar alertas manualmente, únicamente atenderlas o posponerlas.

## Criterios de Aceptación

- El sistema genera una alerta automática al registrar un evento con fecha futura (ejemplo: vacunación próxima).
- Al cumplirse la fecha, la alerta aparece en el tablero principal y permanece hasta que sea marcada como atendida.
- Las alertas pueden visualizarse filtradas por tipo (sanidad, reproducción, alimentación).
- Si un animal sale de la finca, el sistema elimina automáticamente todas sus alertas pendientes.
- El historial del sistema guarda registro de todas las alertas creadas, atendidas y pospuestas, indicando usuario, fecha y acción.
