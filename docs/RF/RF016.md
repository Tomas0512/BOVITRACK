# RF016 — Registro de Eventos Sanitarios

| Campo | Valor |
|-------|-------|
| **Código** | RF016 |
| **Nombre** | Registro de Eventos Sanitarios |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir que el capataz, empleado o el administrador registre todos los eventos relacionados con la sanidad animal, incluyendo: Identificación del animal o lote. Fecha y hora de aplicación de vacunas, desparasitaciones y suplementos vitamínicos. Tipo de vacuna o medicamento aplicado (ej. fiebre aftosa, brucelosis, carbón, antiparasitarios). Dosis administrada y vía de aplicación (oral, inyectable, tópica). Observaciones sobre salud, signos de enfermedad, comportamiento y reacciones adversas. Este registro permite llevar un control sanitario completo del hato, monitorear desparasitaciones y mantener un historial sanitario para trazabilidad y cumplimiento de normativas gubernamentales.

## Controles y Restricciones

- Control de registro por animal o lote: Cada registro debe estar vinculado a animales o lotes previamente registrados.
- Control de consistencia: Las fechas y dosis deben ser coherentes con los protocolos sanitarios vigentes.
- Control de fecha/hora: Cada registro debe incluir fecha y hora para trazabilidad.
- Restricción de permisos: Solo el administrador, capataz o empleado pueden registrar eventos sanitarios; usuarios sin permisos no pueden acceder.
- Restricción de datos obligatorios: No se permite guardar registros incompletos, como identificación de animal/lote, fecha, tipo de vacuna o dosis administrada.

## Criterios de Aceptación

- El sistema permite registrar eventos sanitarios únicamente para animales o lotes válidos.
- Se notifican errores si faltan campos obligatorios o los valores ingresados no son válidos.
- Cada registro incluye fecha, hora, usuario que realizó el registro, tipo de medicamento y dosis administrada.
- Los reportes muestran correctamente el historial sanitario de cada animal o lote, incluyendo vacunaciones, desparasitaciones y observaciones.
