# RF021 — Registro de Ingresos y Salidas de Animales

| Campo | Valor |
|-------|-------|
| **Código** | RF021 |
| **Nombre** | Registro de Ingresos y Salidas de Animales |
| **Versión** | 3.0 |
| **Prioridad** | Alta |

## Descripción

El sistema debe permitir registrar el ingreso y salida de animales en la finca, diferenciando los siguientes casos: Ingresos: nacimientos, compras o traslados desde otras fincas. Salidas: ventas, descartes, muertes o traslados hacia otras fincas. El registro debe contener datos obligatorios como: fecha y hora del evento, identificación individual de los animales, motivo de ingreso o salida, finca o lote de origen/destino, documento soporte (ejemplo: guía de movilización, factura, acta de descarte) y observaciones adicionales. Cada ingreso debe activar al animal en el inventario y cada salida debe dar de baja al animal del inventario activo, manteniendo intacto su historial para trazabilidad.

## Controles y Restricciones

- Control de obligatoriedad: No se podrá guardar el registro si faltan fecha, identificación del animal o motivo del evento.
- Control de unicidad en ingresos: Un mismo animal no puede ser ingresado dos veces a la finca sin que antes haya salido.
- Control de inventario en salidas: El sistema valida que el animal esté activo para poder registrarlo como salida.
- Control documental: El sistema debe exigir adjuntar documento soporte en formatos PDF o imagen cuando se trate de compras o ventas.
- Control de fecha: La fecha de ingreso/salida no puede ser futura ni anterior a la fecha de nacimiento del animal.
- Un ingreso siempre debe estar asociado a una finca y lote existente; no se pueden ingresar animales a una finca no registrada.
- Restriccion de Permisos: Los usuarios sin permisos (empleados no autorizados) no podrán registrar ingresos o salidas.
- Restriccion de Animales: No se podrá dar salida a animales que ya estén inactivos en el sistema.
- Restricción de Registros: El administrador es el único que puede eliminar o anular registros en caso de error.

## Criterios de Aceptación

- Cuando se intente registrar un ingreso con identificación ya existente en la finca, el sistema debe mostrar un mensaje claro de error.
- Al registrar una salida, el sistema debe actualizar en tiempo real el inventario, reflejando la baja del animal en su lote.
- Los reportes de inventario deben conciliar perfectamente la cantidad de animales activos después de cada movimiento.
- El historial del animal debe mostrar de manera cronológica los eventos de ingreso y salida, con sus soportes adjuntos.
- Si falta información obligatoria, el sistema debe impedir guardar el registro y mostrar mensajes específicos (ejemplo: “Debe adjuntar documento soporte”).
- Cada registro debe quedar trazado con usuario, fecha y hora exacta para garantizar la auditoría.
