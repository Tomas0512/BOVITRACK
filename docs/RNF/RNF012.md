# RNF012 — Respaldo y Recuperacion de Datos

| Campo | Valor |
|-------|-------|
| **Código** | RNF012 |
| **Nombre** | Respaldo y Recuperacion de Datos |
| **Versión** | 3.0 |
| **Prioridad** | Baja |

## Descripción

El sistema debe realizar respaldos automáticos de toda la información para evitar pérdida de datos en caso de fallos. Los respaldos deben programarse de manera periódica (ejemplo: cada noche) y también deben poder generarse manualmente. En caso de pérdida de datos, el sistema debe permitir restaurar un respaldo completo.

## Controles y Restricciones

- Control de integridad: Cada respaldo debe incluir todos los datos de las fincas y usuarios.
- Control de seguridad: Los respaldos deben estar encriptados para proteger la información.
- Control documental: Debe existir un registro que indique la fecha, hora y estado de cada respaldo.
- Restricción de permisos: Solo el administrador puede configurar o ejecutar restauraciones.
- Restricción de seguridad: Los respaldos deben almacenarse en un lugar seguro y con clave.
- Restricción funcional: No se pueden restaurar datos de manera parcial; siempre se restaura un respaldo completo.
- Restricción temporal: El sistema solo guarda los respaldos dentro del tiempo de retención definido.

## Criterios de Aceptación

- Los respaldos se realizan automáticamente según la programación y el sistema notifica si hubo un error.
- El administrador puede descargar un respaldo y comprobar que los archivos sean legibles.
- En caso de restauración, toda la información se recupera hasta el último respaldo disponible.
- El sistema guarda un historial con fecha, usuario y estado de cada respaldo.
- Si un respaldo falla, el sistema genera una alerta inmediata al administrador.
